% **1. Theoretical Foundation: Formal Definition of Input and Output**
% *   **Input (Standard RDF Reification):** Let $G_{RDF}$ be an RDF graph containing a reified statement. In standard RDF reification, a statement is structurally represented by an identifier node $i$ of type `rdf:Statement`. This node is linked to the base elements via three core triples: `(i, rdf:subject, s)`, `(i, rdf:predicate, p)`, and `(i, rdf:object, o)`. Metadata attached to this statement is represented by additional triples. To ensure completeness, we distinguish between two types of metadata triples: data properties `(i, p_d, v_d)` where the target is a literal value, and object properties `(i, p_o, n_o)` where the target is another entity. (Hernandez et al. - Reifying RDF: What Works Well With Wikidata?; Cyganiak et al. - RDF 1.1 Concepts and Abstract Syntax; Hogan et al. - Knowledge Graphs)
% *   **Output (Meta-Property Graph):** The target database is a Meta-Property Graph formally defined as the tuple $G_{MPG} = (N, E, P, L, \lambda, \mu, \sigma, \upsilon, \eta, \rho)$. In this structure, $N$ is the set of nodes, $E$ is the set of edges (including directed edges $E_d$), $P$ is the set of properties, and $L$ is the set of label identifiers. The model uses specific mapping functions: $\eta$ for edge topology, $\lambda$ and $\mu$ for labeling, $\sigma$ and $\upsilon$ for property assignment, and the $\rho$ function which natively assigns a set of objects to a specific node to encapsulate a sub-structure. (Sadoughi et al. - Meta-Property Graphs: Extending Property Graphs with Metadata Awareness and Reification)

Map $s$ and $o$ to distinct nodes $n_s$ and $n_o$, and add them to the MPG node set, such that $n_s, n_o \in N$. Use the labeling functions $\lambda$ and $\mu$ to map the predicate name to the edge, such that $p \in \mu(\lambda(e))$.
